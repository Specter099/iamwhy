# PR Template Design

## Context

iamwhy needs a PR template (P2 in repo best practices). Single file at
`.github/pull_request_template.md`.

## Decisions

- **Style**: Focused checklist (5 items), not heavy-weight
- **Sections**: Description, type of change, checklist, breaking changes

## Structure

1. Description — free text (what and why)
2. Type of change — checkboxes (bug fix, feature, breaking, docs, refactor/chore)
3. Checklist — 5 items (tests, passing, lint, coverage, no breaking changes)
4. Breaking changes — optional section
